---
description: Create a structured specification for new features or changes.
---

1.  **Gather Requirements**: Clarify the goal with the user.
2.  **PLANNING Mode**: Switch to `PLANNING` mode.
3.  **Write Spec**: In `implementation_plan.md`, add:
    -   User Stories.
    -   API Endpoints (Methods, Params, Responses).
    -   Database Schema (Migrations).
    -   UI Component Tree / Layout.
    -   Edge Cases.
4.  **Review**: Present the spec to the user via `notify_user`.
5.  **Iterate**: Refine until approved, then proceed to implementation (`/new` workflow).
